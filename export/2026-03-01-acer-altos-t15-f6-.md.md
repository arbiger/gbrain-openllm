## 3. 工具準備：Ventoy SSD (The \"Swiss Army Knife\")

**設備**: **SL Tech 1TB SSD** (集資購買)

**Ventoy 設定參數**:

1.  **安裝模式**: 透過 Windows Ventoy2Disk.exe 安裝。

2.  **保留空間**: 設定保留 **800GB** (Preserve space)。

3.  **分區規劃**:

    - **Partition 1 (Ventoy)**: \~130GB -\> 存放 ISO 檔。

    - **Partition 2 (BACKUP_DATA)**: \~800GB (exFAT) -\> 存放 Clonezilla
      備份檔。

**必備 ISO 清單**:

- proxmox-ve-9.1.iso

- proxmox-backup-server-4.1.iso

- clonezilla-live-3.3.0-33-amd64.iso

- virtio-win.iso (Windows 驅動)

- Windows 11.iso (測試用)

**⚠️ 常見問題 (Troubleshooting)**:

- **Secure Boot 報錯 (Security Violation)**:

  - **方法 A (推薦)**: 進入 BIOS 設定將 **Secure Boot 改為 Disabled**。

  - **方法 B (註冊憑證)**: 若不想改 BIOS，在藍色 MOK 管理畫面選擇 Enroll
    Key from Disk -\> 選擇 VTOYEFI 分區 -\> 選擇
    **ENROLL_THIS_KEY_IN_MOKMANAGER.cer** (此檔案已內建於 USB 中) -\>
    Continue -\> Yes -\> Reboot。

## 5. PBS 備份伺服器架設 (PBS Setup Guide)

**目標**: 建立 VM 101，作為專用備份伺服器。

### 階段 A: 建立與安裝

1.  **上傳 ISO**: 將 proxmox-backup-server-4.1.iso 上傳至 local (pve)。

2.  **Create VM (101)**:

    - OS: PBS ISO。

    - System: 預設。

    - Disk (系統碟): 32GB (於 local-zfs)。

    - CPU: 2 Cores, Type: **Host**。

    - **Memory: 3072 (3GB)** - *節省資源給 ERP*。

3.  **掛載資料碟 (關鍵)**:

    - VM 建立後 **勿開機**。

    - Hardware -\> Add -\> Hard Disk。

    - Storage: **backup-pool**。

    - Size: **1600** GB (保留空間給 ZFS)。

    - **⚠️ Exclude Backup**: 進入 Edit，**取消勾選 Backup**
      (避免備份到備份碟)。

4.  **安裝系統**:

    - 啟動 VM -\> Install。

    - Target Disk: 務必選 **32GB** 那顆。

    - IP: 192.168.0.232 (或測試段)。

### 階段 B: 初始化與連線

1.  **初始化 Datastore**:

    - 瀏覽器登入 PBS (:8007)。

    - Storage / Disks -\> Initialize 1.6TB Disk (GPT)。

    - Directory -\> Create: Name **backup-store**, Filesystem xfs.

2.  **PVE 連接 PBS**:

    - PBS Dashboard -\> 複製 **Fingerprint**。

    - PVE -\> Datacenter -\> Storage -\> Add PBS。

    - ID: PBS-232 / Server: IP / Datastore: backup-store。

    - *故障排除*: 若 GUI 報錯 (500)，使用 Shell 指令強制連線：\
      proxmox-backup-client list \--repository root@pam@IP:backup-store\
      \# 輸入 yes 接受指紋

### 階段 C: 備份排程

1.  **Datacenter** -\> Backup **-\> Add**。

2.  Schedule: Daily (03:00)。

3.  Selection Mode: Selected VMs (手動勾選 ERP VM)。

4.  Retention: Keep Last 7, Daily 14.

## 6. P2V 轉移流程詳解 (Clonezilla Migration Detail)

**情境**: 將舊 ERP Server (Win 2008 R2) 備份並還原至 PVE VM。

**工具**: SL Tech SSD (Ventoy + BACKUP_DATA 分區)。

### 第一部分：備份篇 (Dump)

1.  **開機與設定**:

    - 插入 SSD，舊機開機選 USB。

    - 選擇 clonezilla-live-3.3.0-33-amd64.iso。

    - 語言: zh_TW / 鍵盤: Keep / Start_Clonezilla。

2.  **掛載 (Mount) - 關鍵步驟**:

    - 選擇 device-image -\> local_dev。

    - 插入裝置按 Enter -\> 等待偵測按 Ctrl+C。

    - **選擇掛載點**: 選取容量約 **800GB** 的那個分區 (BACKUP_DATA)。

    - 目錄選擇 /。

3.  **執行備份**:

    - 模式: Beginner -\> 功能: savedisk。

    - 命名: 輸入備份檔名 (如 2026-01-ERP-Backup)。

    - **來源**: 選取本機硬碟 (sda/nvme0n1) -\> 按空白鍵勾選。

    - 壓縮/檢查: 預設 -\> Yes (檢查) -\> No (不加密)。

    - 確認執行 (輸入 y)。

4.  完成後關機 (Poweroff)。

5.  **P2V 舊機遷移標準程序 (Migration SOP) - 專家版**

> 適用對象: 舊 Dell T110 (Windows Server 2008 R2, Legacy BIOS)。
>
> 工具: Clonezilla ISO, 外接 SSD。
>
> 階段 I: 來源端備份 (Backup with Rescue)

- Clonezilla 模式: device-image -\> savedisk。

- 關鍵參數 (針對壞軌舊機):

- 進入 Expert (專家) 模式。

- 勾選 -rescue (Continue reading next one when disk blocks read
  errors)。

- -sfsck (Skip checking source fs)。

- 執行: 等待備份完成，確認 log 沒有 fatal error。

### 第二部分：還原篇 (Restore)

1.  **VM 準備 (關鍵: 容量設定)**:

    - PVE 建立新 VM (ID 100)。

    - **Hard Disk 設定**:

      - ⚠️ **容量規則**: 虛擬硬碟 **必須 \>=** 來源實體硬碟大小。

      - **Discard**: 勾選。

    - **Hardware**: Add USB Device (Passthrough SL Tech SSD)。

    - **CD/DVD**: 掛載 Clonezilla ISO。

    - **Options**: Boot Order 設光碟優先。

### 目的端 VM 設定 (PVE Target)

- 建立一個能讓 Windows 2008 R2 跑起來的「完美空殼」：

  --------------------------------------------------------------------------
  **選項**      **設定值**          **關鍵理由**
  ------------- ------------------- ----------------------------------------
  **OS Type**   Windows 2008r2/7    優化舊核心參數

  **BIOS**      **SeaBIOS**         T110 是 Legacy BIOS，選 UEFI 會開不了機

  **CPU**       **host** (1 Socket, 透傳 Xeon 指令集，效能最佳
                4 Cores)            

  **Memory**    **16384 MB (16GB)** 對應舊機規格

  **Hard Disk** Bus: **IDE**        **避開 0x7B 藍屏的唯一解** (2008 R2 無
                (512GB)             VirtIO 驅動)

  **Cache**     **Write back**      加速還原寫入速度 (依賴 UPS 保護)

  **Network**   **Intel E1000**     2008 R2 內建驅動，保證有網
  --------------------------------------------------------------------------

- 

2.  **開機與掛載**:

    - 啟動 VM 進入 Clonezilla，操作同上。

    - **掛載點**: 選擇 Passthrough 進來的 **800GB USB 分區**。

3.  **執行還原**:

    - 模式: Beginner -\> 功能: **restoredisk**。

    - 映像檔: 選擇剛剛的備份檔。

    - **目的**: 選擇 VM 的虛擬硬碟 (vda/sda)。

    - 確認執行 (輸入兩次 y)。

4.  完成後關機 (Poweroff)。

5.  Clonezilla 還原 (Restore with MBR Fix)

    - 掛載 Clonezilla ISO 與 USB SSD 至 VM，開機進入 Console。

    - 還原模式: 選 restoredisk。

    - 關鍵參數 (專家模式):

    - 勾選 -j2 (Clone hidden data between MBR and 1st
      partition)。這是修復 \"No bootable device\" 的關鍵。

    - 執行:

    - 務必等待進度條跑完後，出現 Restoring the first 446 bytes of
      MBR\... 和 Finished 字樣。

    - 切勿提早強制關機。

### 第三部分：藍屏救援與驅動安裝 (BSOD Rescue)

**狀況**: 若開機出現 INACCESSIBLE_BOOT_DEVICE 藍屏，請執行
**雙硬碟誘騙法 (The Dummy Disk Trick)**。

**步驟 A: 準備工作 (保持 SATA 開機)**

1.  **Stop** VM。

2.  **Hardware** -\> 選擇 Hard Disk (scsi0) -\> **Detach**。

3.  編輯 Unused Disk 0 -\> 改 Bus 為 **SATA** -\> **Add**。

4.  **Options** -\> Boot Order -\> 勾選 sata0 為第一位。

5.  **Start** VM -\> 確認能進入 Windows 桌面。

**步驟 B: 誘發驅動 (新增 SCSI 誘餌)**

1.  保持 VM 開機 (或關機做也可以)。

2.  **Hardware** -\> **Add** -\> **Hard Disk**。

3.  **Bus/Device**: 務必選 **SCSI**。

4.  **Size**: 1 GB (小容量即可)。

5.  **Add**。

6.  若 VM 開著，Windows 會偵測新硬體；若關著，請開機進入桌面。

7.  **檢查**: 裝置管理員 -\> 儲存控制器 -\> 確認出現 **Red Hat VirtIO
    SCSI controller** (無驚嘆號)。

**步驟 C: 最終切換 (Final Switch)**

1.  **Shutdown** Windows。

2.  **Hardware**:

    - Detach 並 Remove 那顆 1GB 誘餌碟。

    - Detach 主硬碟 (sata0)。

    - Edit Unused Disk 0 -\> Change Bus to **SCSI** -\> Add.

3.  **Options** -\> Boot Order -\> Check scsi0 為第一位。

4.  **Start** VM -\> **成功開機！**

### 🔄 ERP 內部備份自動清理腳本 (Auto Delete Old Backups)

**目的**: 自動刪除 E:\\上華備份 資料夾中超過 7 天的 .bak 舊備份檔。

**部署位置**: ERP Windows Server (VM 100)

**腳本內容 (請另存為 AutoDeleteERPBackup.bat，編碼 ANSI)**

\@echo off\
set BACKUP_PATH=E:\\上華備份\
set DAYS=7\
echo 正在檢查 %BACKUP_PATH% 超過 %DAYS% 天的 .bak 舊檔案\...\
forfiles /p \"%BACKUP_PATH%\" /s /m \*.bak /d -%DAYS% /c \"cmd /c del
\@path\"\
echo 清理完成!

**設定排程**:

1.  將 .bat 檔放在 C:\\Scripts\\。

2.  開啟工作排程器 -\> 建立基本工作 (Daily)。

3.  **關鍵設定**: 在「開始位置 (Start in)」欄位，填入 C:\\Scripts\\。

## 8. 搬遷前 IP 切換標準程序 (Migration SOP)

**策略**: 優先確保 PVE 主機可連線，其餘 VM 抵達新現場後再透過 Console
修改。

**本次變更目標**：

- **PVE**: 192.168.1.231 -\> **192.168.0.231**

- **PBS**: 192.168.1.232 -\> **192.168.0.232**

- **Gateway**: 192.168.0.1

## 階段 1: 修改 PBS IP (在舊網段操作)

### 注意：這步做完後，PBS 會暫時斷線，直到 PVE 也搬到新網段為止。

1.  登入 PBS 的 Web 介面 (或透過 PVE Console 進入)。

2.  進入 Configuration (設定) -\> Network/Interfaces (網路)。

3.  選取網卡 (通常是 eth0 或 ens18)，點擊 Edit (編輯)。

    - IPv4/CIDR: 改為 192.168.0.232/24

    - Gateway: 改為 192.168.0.1

4.  點擊 Apply Configuration (套用設定)。

    - 此時 PBS 網路會斷開，這是正常的。

## 階段 2: 修改 PVE IP (透過 Web UI 操作的關鍵步驟)

**警告：請嚴格遵守順序，先改 Hosts，再改 Network。**

1.  步驟 A：修改 Hosts (最重要！)

    - 在 PVE Web 介面，點擊左側節點 (pve)。

    - 選擇 System (系統) -\> Hosts。

    - 找到舊 IP 192.168.1.231 的那一行，點擊編輯。

    - 將 IP 改為 192.168.0.231。

    - 存檔。

2.  步驟 B：修改 Network

    - 選擇 System (系統) -\> Network (網路)。

    - 選取 vmbr0，點擊 Edit (編輯)。

    - IPv4/CIDR: 改為 192.168.0.231/24

    - Gateway: 改為 192.168.0.1

    - 點擊 OK。

3.  步驟 C：套用並搬家

    - 點擊上方的 Apply Configuration (如果有的話)，或者直接將 PVE 關機
      (Shutdown)。

    - 拔線移機：將伺服器搬到新環境，接上新路由器的網路線。

    - 開機。

## 救命方案：如果連不上怎麼辦？(Direct Connect)

如果開機後發現 192.168.0.231 連不上：

1.  找一台筆電，用網路線直接連接 Acer Altos T15 的網孔。

2.  筆電網路設定：

    - IP: 192.168.0.99 (隨便一個同網段沒人用的)

    - Mask: 255.255.255.0
