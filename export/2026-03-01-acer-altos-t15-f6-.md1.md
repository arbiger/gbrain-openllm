## 1. 硬體盤點與儲存策略 (Infrastructure)

- **伺服器**: Acer Altos T15 F6 (Tower Server)

- **CPU**: **AMD EPYC 4344P** (8 Cores / 16 Threads, Zen 4 Architecture)

- **RAM**: **32GB DDR5-4800 ECC UDIMM** (16GB x 2)

- **儲存配置**:

  - **系統池 (Boot Pool)**: 2TB M.2 NVMe SSD x 2 -\> 組成 **ZFS Mirror
    (RAID 1)**，安裝 PVE OS。

  - **備份池 (Backup Pool)**: 2TB SATA HDD x 2 -\> 用於 PBS (Proxmox
    Backup Server) 儲存。

## 2. BIOS 關鍵設定 (BIOS Setup)

**警告**：重置 BIOS 後務必檢查此區，否則虛擬化與 ZFS 將失效。

1.  **Virtualization**:

    - SVM Mode (AMD): **Enabled** (開啟虛擬化支援)

    - IOMMU: **Enabled** (開啟硬體直通支援)

2.  **Storage**:

    - SATA Mode: **AHCI** (讓 ZFS 直接控制硬碟，**切勿**開啟 RAID
      Card/RST)

3.  **Boot**:

    - Boot Mode: **UEFI**

    - Secure Boot: **Disabled** (避免驅動簽章問題)

## 3. 系統安裝標準程序 (Installation SOP)

1.  **準備**: 製作 Ventoy USB，放入 proxmox-ve_x.x.iso。

2.  **安裝 PVE**:

    - **Target Disk**: 選擇兩顆 NVMe SSD。

    - **Filesystem**: 選擇 zfs (RAID1)。

3.  **安裝 PBS (作為 VM)**:

    - 建議將 PBS 安裝在 PVE 內的 VM，並將 SATA HDD 直通給它，或掛載 ZFS
      Dataset。

## 4. 網路與內網穿透 (Tailscale Network)

**目標**: 不依賴固定 IP 與防火牆開 Port，實現全球安全存取。

1.  **建立 LXC**: 使用 ubuntu-22.04 模板，規格 1 Core / 512MB RAM。

2.  **開啟轉發 (在 LXC 內)**:\
    編輯 /etc/sysctl.conf，啟用 net.ipv4.ip_forward = 1。

3.  **安裝 Tailscale**:\
    curl -fsSL
    \[https://tailscale.com/install.sh\](https://tailscale.com/install.sh)
    \| sh

4.  **啟動路由廣播**:\
    tailscale up \--advertise-routes=192.168.0.0/24 \#
    注意網段需配合當前環境

5.  **後台放行**: 至 Tailscale Admin Console 啟用 (Enable)
    該路由，並關閉 Key Expiry。

## 5. P2V 舊機遷移標準程序 (Migration SOP) - 詳細版

**工具**: Clonezilla (再生龍) ISO (放入 Ventoy)。

### 階段 I: 來源端備份 (舊機 -\> 映像檔)

1.  **啟動 Clonezilla**: 選擇語言 (zh_TW) -\> 鍵盤預設 -\> Start
    Clonezilla。

2.  **選擇模式**: 選 **device-image** (存成映像檔)。

3.  **掛載儲存區 (Repository)**:

    - 選 **local_dev** (存到外接 USB 硬碟)。

    - 插入 USB 硬碟，等待偵測後按 Enter。

    - 選擇對應的分割區 (例如 sdb1) 掛載到 /home/partimg。

4.  **執行備份**:

    - 模式: **Beginner** (初學者)。

    - 動作: **savedisk** (備份整顆硬碟)。

    - 命名: 輸入易辨識的名稱 (如 dell-server-backup)。

    - 來源: 按空白鍵選取舊機系統碟 (sda)。

5.  **關鍵選項 (加速與容錯)**:

    - 壓縮: 預設 z1p 或 z9p。

    - **檔案系統檢查**: 選 **-sfsck (Skip /
      不檢查)**。*(極重要，節省大量時間)*

    - **映像檔檢查**: 建議選 Yes (備份後驗證)。

    - 加密: not-encrypt。

6.  **壞軌處理 (緊急救援)**:

    - 若備份過程中出現 Bad sector 紅字錯誤。

    - **處置**: 按下 Enter 或輸入 y 選擇 **Continue/Skip
      (忽略並繼續)**。

    - *說明:
      這是為了先救出剩餘資料。還原後可能會有少量檔案損毀，需進行後續修復。*

### 階段 II: 目的端還原 (映像檔 -\> PVE VM)

1.  **建立 VM**: 在 PVE 建立新 VM，硬碟容量需 **\>= 舊機硬碟**。

2.  **掛載**: 將 Clonezilla ISO 掛入 CD-ROM，並設定優先開機。

3.  **還原**: 進入 Clonezilla -\> device-image -\> 掛載剛剛的 USB -\>
    restoredisk。

4.  **目標**: 選擇 VM 的虛擬硬碟 (vda 或 sda)。

### 階段 III: 異機還原修復 (Post-Migration Fix)

1.  **Windows**:

    - 藍屏 (INACCESSIBLE_BOOT_DEVICE): 掛載
      virtio-win.iso，進入修復模式安裝驅動，或改用 IDE/SATA 模式開機。

    - 磁碟檢查: 開機後務必執行 chkdsk /f 修復壞軌造成的邏輯錯誤。

2.  **Linux**:

    - 修正 /etc/fstab: UUID 可能改變，需更新。

    - 修正網卡: /etc/netplan 或 /etc/network/interfaces 中的網卡名稱 (如
      eth0 -\> ens18)。

    - 磁碟檢查: touch /forcefsck 並重開機。

## 6. 電源保護策略 (UPS Setup)

**設備**: **CyberPower CP1500PFCLCDa** (1500VA/1000W)

**連接**: USB Type-B 連接至 PVE 主機。

1.  **安裝 NUT**: apt install nut

2.  **設定驅動 (/etc/nut/ups.conf)**:\
    \[myups\]\
    driver = usbhid-ups\
    port = auto\
    desc = \"CyberPower UPS\"

3.  **設定監控 (/etc/nut/upsmon.conf)**:\
    MONITOR myups@localhost 1 upsmon pass master

4.  **啟動服務**: service nut-server start, service nut-client start。

5.  **驗證**: upsc myups 應顯示電池狀態。

6.  **斷電測試**: 務必實際拔掉市電，確認 PVE 會收到 FSD 訊號並自動關機。

## 7. 移機與 IP 變更 SOP (Relocation Guide) - 詳細版

**情境**: 網段從 192.168.1.x 變更為 192.168.0.x。

**Gateway**: 192.168.0.1

**順序**: **先改 PBS (VM) -\> 再改 PVE (Host) -\> 最後修復連線。**

### 階段 I: 修改 PBS IP (在舊網段操作)

1.  進入 PBS Web UI 或 Console。

2.  **Configuration -\> Network/Interfaces**:

    - 編輯 eth0。

    - IP 改為 192.168.0.232/24。

    - Gateway 改為 192.168.0.1。

3.  點擊 **Apply Configuration** (此時 PBS 會斷線，正常)。

### 階段 II: 修改 PVE IP (透過 Web UI 操作)

**警告：務必遵守順序：先 Hosts，後 Network**

1.  **修改 Hosts (避免 Web UI 掛掉)**:

    - PVE 介面 -\> **System -\> Hosts**。

    - 將舊 IP 192.168.1.231 改為 192.168.0.231。

    - 存檔。

2.  **修改 Network**:

    - PVE 介面 -\> **System -\> Network**。

    - 編輯 vmbr0。

    - IP 改為 192.168.0.231/24。

    - Gateway 改為 192.168.0.1。

    - 點擊 OK。

3.  **移機**:

    - 關機 PVE。

    - 搬運至新地點，接上新路由器網路線。

    - 開機。

### 階段 III: 上線與修復

1.  在新地點使用新 IP https://192.168.0.231:8006 登入。

2.  **修復 Storage 連線**:

    - PVE 介面 -\> **Datacenter -\> Storage**。

    - 編輯 PBS 儲存區。

    - **Server**: 將舊 IP 改為 192.168.0.232。

    - 存檔，確認狀態變亮。

3.  **修復 Tailscale**:

    - 進入 Tailscale LXC。

    - 執行 tailscale up \--advertise-routes=192.168.0.0/24 \--reset。

    - 至 Tailscale Admin Console 重新放行路由。

### 救命方案 (Emergency Access)

若無法連線：

1.  筆電直連 PVE 網孔。

2.  筆電設固定 IP 192.168.0.99 / Mask 255.255.255.0。

3.  瀏覽器開啟 192.168.0.231:8006 進行除錯。
