# 让AI变成Agent只需要给它充分的自主性

---

Hello，下班了。今天我在北京下班，啊，今天跟大家分享一下我用这个 AI agent 哈，就是怎么启动一个 agent 这个事情呢？是我前两天做梦梦到的，很神奇，是不是？因为最近一段时间这个 Cloud Bot 非常火，什么原理呢？就是给 AI 一个电脑，给它充分的自主性。 让他去开始在电脑里面去探索，然后执行任务。他因为有了手和脚，所以他就可以去完成指定的任务。前两天我就突然间做了一个梦，我就说这个怎么才能让AI开始自己干活呢？因为我之前分享说，我们有同事已经开始指挥agent，然后让他指挥下面的sub agent，让他们做很多的工作。 他已经把自己升级成G M了，就是总经理了，对吧？他只要管好那个agent作为总监的，他就可以高效的完成很多工作。然后我就很好奇，他到底是怎么让让这个agent开始自主干活的？我一直很好奇。我那天那个梦呢，说的很神奇哈，就是真的我是梦到了这一点。那个梦里面就解决了一个问题。我前两天在发烧啊，所以就是半梦半醒的状态下，突然就明白了一件事儿，就是当你给了AI自主性之后，它就可以开始干活。 啥叫自主性呢？我们说 prompt 的是人给他的一个要求，然后他就可以基于这个要求就开始干活了。那如果你给他的这个要求是一个非常复杂的任务，或者说是一个非常长程的任务，那他就可以开始在你的电脑里面去执行一个复杂的动作。因为我之前的我们叫 Codebarry 是吧？就是我们一个 IDE AI 的 IDE，它在这里面它可以调用很多工具，它可以读我电脑中的文件，然后它可以写文件。 它其实已经开始可以执行了，就像 cloud code 一样，或者就像这个 cloud bot 一样，就它已经匹配了很多的工具和能力，只是说目前我们没有给它充分的自主性，让它去探索。假设说，我给它自主性，那它就可以开始干活了。我举个例子，我们今天呢在开会，各种各样的会哈，然后我就跟 AI 说了一句话，我说，我现在没有时间跟你讨论非常详细的这个问题，但是呢，我有一个课题一直放在那儿没有推进。 因为一些原因，然后你现在你来作为这个项目的PM，帮我开始做这个项目。你找我要资源，就是你觉得缺什么，你可以问我，但是我现在没有时间回复你。你可以写一个文档，把需要我解答的一些内容写到这个文档中，然后你自己开始探索吧。然后我就放手了，我给了他自主性，然后他就可以开始干活了。他就一个一个AI在那跑跑跑跑了好久的时间。哎呀，我走的这个方向对不对呀？ 啊，就这样吧。对，然后他就开始干活了。这、这、这、这个就是一个 agent 形态呀。其实我那同事他让 AI 在他电脑里面去探索，这个不就是让他自由探索，就是给他授权，让他让他开始干活吗？不就是 agent 吗？没什么复杂的。其实就是我以前可能把这件事情想象的太复杂了。就是你觉得要有。 一套什么样的标准，一个什么样的skills，一个什么样的一个机制激活它，但实际上没有那么复杂，就是一句话：可充分的授权。为什么这次的这个Cloud Bot会这么火，会大家觉得很意外？就是因为你当你给AI一个电脑、一些工具，然后无限的token，然后以及一个任务方向的时候，它就可以开始执行、开始干活。其实我们现在已经可以实现了，就是在Code Buddy里，在Cloud Code里面，当你的任务足够具体，然后AI这个。 授权的范围足够大的时候，它就是已经可以开始去作为 agent 去执行了。只不过我们现在的任务没有那么复杂，的就需要它工作五个小时以上。我们实际上大部分的任务都没有那么复杂。所以今天下午在跟 AI 交互的过程中，我就去跟他说：“我说，你你你这个，你再想一想，对吧？因为我会还没有开完。我跟他说，你换个角度，你站在老板的视角再考虑一下这个问题，你觉得需要什么样的资源去完成这件事儿？你可以做更充分的探索之后再跟我说答案。” 你先不要着急跟我回复，你先自己探索一下，你有充分的去了解现在市场的情况吗？你回头你了解了之后，你回来再看一下，你说的对不对？就基本上，当你把自己变成一个管理者的时候，那你的AI就是可以听命于你，就帮你干活的，对不对？那我今天的体验就是这个，所以我觉得大家可以试起来，就是可以暂时先投入一些钱在token的这个消耗上，我觉得不会太多，但是呢，你会发现它可以做很多事情。昨天听那个欧耶那个播客。 主持人就说，当他把他电脑装上这个 Cloud Bot 的时候，他就发现，它可以代表
