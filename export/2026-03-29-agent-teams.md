# Agent Teams 設計的兩種思路

---

## 摘要

這部影片探討了 **Agent Teams（智慧體小隊）編排設計**的兩種不同思路，分別適用於「確定性編程場景」與「不確定性研究場景」。

### 🏭 場景一：確定性編程場景（Anthropic 風格）

- 需要明確架構：有一個 PM（產品經理）負責組織整個 Agent 協作
- 設立明確驗證機制：判斷方向是否正確、如何激勵
- 執行 Agent 之間**不需要太多討論**，執行就好
- 用強大模型（如 Open 4o）作為大腦做規劃決策，底下用較快的 Agent（如 Haiku）執行
- 典型案例：用 Multi Agent 寫 C 編譯器

### 🔬 場景二：不確定性研究場景（晓辉博士的實踐）

**與編程場景的共同點：**
- 透過文件交互讓 Agent 寫入發現，下一個 Agent 可讀取
- 需要 PM 控制研究進度
- 需要審核機制

**研究場景特別需要的三個關鍵角色：**

1. **批判者（Critic）** 🧠
   - 做研究需要批判性思維，站在第一性原理持續反思假設是否成立
   - 批判者在研究結果的品質提升上效果顯著
   - AI 傾向自圓其說，批判視角能帶來對抗性思考

2. **探索者（Explorer）** 🔭
   - 不在既有文獻圈圈裡打轉
   - 研究是開放性的，需要去找新的視角和機會
   - 如同團隊預留 20% 探索時間的原則

3. **Agent 間的討論機制** 💬
   - 編程場景避免過多討論（會引入雜訊）
   - 研究場景**需要討論**，因為學科間有 Gap，討論能激發新思考
   - 面對不確定性問題，Agent 間的通信變得非常重要

### 總結

| 維度 | 編程場景 | 研究場景 |
|------|---------|---------|
| 通訊 | 避免討論，執行就好 | 需要討論激發新思考 |
| 批判者 | 不需要 | 非常需要 |
| 探索者 | 不需要 | 需要 |
| 模型策略 | 大腦用強模型，執行用快模型 | 大腦 + 各領域專家 |
