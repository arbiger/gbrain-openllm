# Claude Code 六维记忆体系深度解析

> **TLDR:** Claude Code 构建了一套六维记忆体系（指令记忆→短期记忆→工作记忆→长期记忆→摘要级记忆→休眠重塑记忆），系统性解决了长程对话上下文遗忘、多端同步和 token 预算控制三大核心问题。其中 Auto Dream 机制可在后台自动整理碎片化记忆，是当前 AI 编程工具中最完整的记忆架构设计。

## 2. 六维记忆体系详解

### 第一层：指令记忆（规则级记忆）
**核心载体:** `claude.md` 文件组（多层级静态指令加载系统）

**四层优先级（从低到高）：**
| 层级 | 路径 | 范围 |
|------|------|------|
| 1 | `/etc/claude-code/claude.md` | 全局策略（组织管理员设定） |
| 2 | `~/.claude/claude.md` | 私有全局指令（跨所有项目） |
| 3 | `./claude.md`（项目根目录） | 团队共享的项目级指令 |
| 4 | `./claude.local.md` | 私有项目指令（不纳入版本控制） |

**加载机制:** 从根目录向 cwd 遍历，离工作目录越近的规则优先级越高

**三种扩展机制:**
1. `@include` 递归包含（最多5层，有循环检测）
2. 条件规则（可在 `.claude` 目录中组织）
3. 目录级配置（子目录可有独立的 `.claude`）

### 第二层：短期记忆
- 当前会话的完整对话历史
- 全部存在内存中，无压缩

### 第三层：工作记忆
- 与当前执行任务相关的动态状态
- 包含：任务进度、偏移量、投机响应状态等

### 第四层：长期记忆（核心部分）
- 基于磁盘 Markdown 文件构建的分层知识库引擎
- 在用户目录下创建专用记忆文件夹
- **四种知识类型:**
  - 用户画像
  - 行为反馈
  - 项目上下文
  - 外部引用

### 第五层：摘要级记忆（压缩记忆）
- 当 Token 快爆时触发压缩
- 引入 **Session Memory 机制**：通过后台子代理持续维护一份会话笔记
- 压缩时直接用这份笔记替代临时生成的摘要（而非让 LLM 重新生成摘要）

### 第六层：休眠重塑记忆（Auto Dream）
- 当满足「一定时间和绘画次数条件」后自动启动
- 执行「碎片化记忆清理、合并、去除矛盾」流程
- 更新全局索引
- **名称隐喻:** 像人类做梦一样在后台自动整理记忆

## 3. 关键设计亮点

| 设计 | 说明 |
|------|------|
| 多层优先级系统 | 从全局到项目到个人的分层指令体系 |
| 目录遍历加载 | 从根到 cwd 的顺序确保局部规则优先 |
| Session Memory | 后台子代理持续维护会话笔记，避免临时生成摘要 |
| Auto Dream | 满足条件后自动整合碎片化记忆 |
| 知识分类体系 | 用户画像/行为反馈/项目上下文/外部引用四分法 |

---

## 4. 对 Skills / Agent 开发的启示

1. **分层记忆架构**: 将记忆分为「规则层（指令）」「工作层（短期）」「知识层（长期）」「压缩层（摘要）」「整合层（休眠）」，每层有明确的职责边界
2. **后台子代理维护**: Session Memory 由后台子代理持续维护，而非临时调用 LLM 生成
3. **自动整合机制**: Auto Dream 展示了一种「低优先级后台任务」的设计模式——不干扰主流程，但在合适时机主动整理
4. **条件触发**: 六层记忆的触发都有明确条件（对话长度、时间、绘画次数等），而非全量加载
