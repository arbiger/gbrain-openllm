# 為何95%的AI Agent做不起來（2025年11月架構師觀察）

---

【2025年11月 Tweet - AI Agent 為何 95% 做不起來】

核心觀點：

1. **Context Engineering > Prompt Engineering**
現階段大家都還在拼 prompt，少數人開始拼上下文結構。真正落地的 Agent，都在「什麼該讓模型看、怎麼看、以什麼形式看」上下足了功夫。

2. **記憶系統要分層**
很多公司的 memory 只是聊天記錄倉庫。真正需要：使用者級 / 團隊級 / 系統級分層記憶，還要能讓人知道「它記住了什麼」，使用者能自己改。否則不是記憶，是監控。

3. **多模型 Routing 是標配**
一個 Agent 後面綁定的是一個 LLM 團隊：輕模型做分類和前處理、重模型做主任務、另一個做驗證或追問。

4. **可追溯 / 可控 / 可信是企業底線**
企業關心：這句話從哪裡來的？有沒有越權？出了錯怎麼追責？AI 要可治理。

5. **Agent ≠ Chatbot**
還在用聊天當所有用戶互動方式，Agent 最多只是語音助理。真正的 Agent 應該：先用語言調度任務，在頁面上看到結構化結果，還能繼續點選、調整、組合下一步。

【有趣的細節】
當主持人問觀眾「你們中有多少人構建了文本到 SQL 並投入生產？」時，沒有一個人舉手。
