## Step 2 補充：tmux Session 啟動方式

**tmux vs. codex/claude 直接 exec**

作者之前用 `codex exec` 或 `claude -p`，最近改用 tmux。
原因：mid-task redirection 很強大。Agent 走向錯誤方向？不用 kill，直接重新導向：

```bash
tmux send-keys -t codex-templates "Stop. Focus on the API layer first, not the UI." Enter
```

**實際啟動指令：**

Codex：
```bash
codex --model gpt-5.3-codex \
 -c "model_reasoning_effort=high" \
 --dangerously-bypass-approvals-and-sandbox \
 "Your prompt here"
```

Claude Code：
```bash
claude --model claude-opus-4.5 \
 --dangerously-skip-permissions \
 -p "Your prompt here"
```

tmux session 範例：
```bash
tmux new-session -d -s "codex-templates" \
 -c "/Users/elvis/Documents/GitHub/medialyst-worktrees/feat-custom-templates" \
 "$HOME/.codex-agent/run-agent.sh templates gpt-5.3-codex high"
```

每個 agent 有獨立 worktree（隔離 branch）+ 獨立 tmux session，完整 terminal logging。
